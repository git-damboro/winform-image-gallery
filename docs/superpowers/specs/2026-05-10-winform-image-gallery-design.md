# WinForm 图像浏览程序设计

## 背景

在 `E:\Desktop\Mryao\WinForm` 新建一个基于 .NET 6 或更高版本的 WinForms 图像浏览程序。程序需要以缩略图方式展示大量图片，支持添加、删除、多选、滚动、缩略图尺寸调整、大图预览和多种显示风格，其中 `Crystal` 为自绘水晶质感缩略图卡片风格。

## 目标

- 显示多张图片，按行列排列缩略图。
- 缩略图下方显示文件名、文件大小、图像类型等属性。
- 支持添加单张或多张图片。
- 支持单选删除和多选删除，删除仅从浏览列表移除，不删除磁盘文件。
- 图片数量超过窗体区域时自动显示水平和垂直滚动条。
- 使用滑动条调整缩略图大小。
- 鼠标悬停或双击缩略图时显示大图，鼠标移开后自动关闭。
- 支持 `Default`、`Compact`、`Crystal` 等显示风格。
- 支持 1 万张图片、超大图和主流图像格式，保持较低内存占用和较强健壮性。

## 非目标

- 不做图片编辑功能。
- 不删除磁盘上的源图片文件。
- 不引入第三方 UI 主题库实现 `Crystal`，该风格由程序自绘。
- 不在首版稳定承诺 HEIC、AVIF 等需要额外解码库的格式。

## 架构

采用自绘虚拟化缩略图控件，而不是为每张图片创建 `PictureBox` 和 `Label`。这样可以避免 1 万张图片对应 1 万组 WinForms 控件导致的卡顿和高内存占用。

核心模块：

- `MainForm`：主窗体，承载工具栏、添加/删除按钮、缩略图大小滑动条、风格选择框和图像区域。
- `ImageGalleryControl`：自定义控件，负责虚拟化布局、滚动、选择、鼠标交互和绘制。
- `ImageItem`：图片元数据模型，保存路径、文件名、文件大小、扩展名、图片尺寸、加载状态和错误信息。
- `ImageFileService`：处理文件筛选、批量添加和元数据读取。
- `ThumbnailService`：后台异步生成缩略图，限制并发并维护 LRU 缓存。
- `GalleryRenderer`：根据显示风格绘制缩略图卡片、文本和选中态。
- `PreviewForm`：显示悬停或双击触发的大图预览。

## 布局与交互

`ImageGalleryControl` 根据当前缩略图尺寸、文本区域高度、间距和控件可视区域计算列数、行数和总内容尺寸。绘制时只计算并渲染当前滚动区域可见的项目索引。

选择规则：

- 单击缩略图：清除原选择并选中当前项。
- `Ctrl + 单击`：切换当前项选择状态。
- `Shift + 单击`：按范围选择。
- 删除按钮：删除当前所有选中项。

预览规则：

- 鼠标悬停到缩略图上后显示预览窗体。
- 鼠标离开缩略图或预览区域后关闭预览。
- 双击缩略图立即显示预览。
- 预览加载失败时显示错误占位内容，不抛出未处理异常。

## 显示风格

- `Default`：标准浅色卡片，清晰边框和选中态。
- `Compact`：更小间距和更紧凑的文字区域，适合大量图片快速浏览。
- `Crystal`：水晶质感卡片，自绘高光、浅色渐变、细边框、轻阴影和选中发光效果。

## 性能策略

- 添加图片时只读取文件路径和轻量元数据，不立即解码全部图片。
- 缩略图按需生成，仅为可视区域和少量预取区域生成。
- 缩略图缓存有固定上限，采用 LRU 淘汰并及时释放 `Image` 对象。
- 后台缩略图生成限制并发，避免拖慢 UI 线程。
- UI 线程只负责绘制和响应交互，文件 IO 和解码在后台执行。
- 超大图加载时优先生成目标尺寸缩略图，避免长期持有原图大对象。

## 格式与异常处理

首版支持 Windows/GDI+ 常见格式：`.jpg`、`.jpeg`、`.png`、`.bmp`、`.gif`、`.tif`、`.tiff`、`.ico`。如果后续需要稳定支持 `.webp`、`.heic`、`.avif`，再引入 `SixLabors.ImageSharp` 或 `Magick.NET`。

异常处理覆盖：

- 文件不存在或被移动。
- 权限不足。
- 文件被占用。
- 图片损坏或格式不支持。
- 超大图解码失败。
- 后台任务取消或队列拥塞。

所有单张图片错误只影响该图片项，显示错误占位图和提示，不导致程序退出。

## 测试策略

优先测试非 UI 逻辑：

- 文件格式筛选。
- 文件大小格式化。
- 虚拟化布局计算。
- 可见索引范围计算。
- 多选删除逻辑。
- 缩略图缓存淘汰逻辑。

WinForms 绘制和预览交互做最小人工验证，确保添加、滚动、缩放、选择、删除、悬停预览和 `Crystal` 风格可用。

## 成功标准

- 能在 .NET 6 或更高版本下构建运行。
- 能添加单张和多张图片。
- 能显示文件名、大小和类型。
- 能通过滑动条调整缩略图大小。
- 能单选和多选删除列表项。
- 能在 1 万张图片元数据列表下保持滚动响应。
- 图片解码失败不会导致程序崩溃。
- `Crystal` 风格可被选择并呈现明显不同的水晶质感。
